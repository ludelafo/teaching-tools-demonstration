## Titre de niveau 2

Je peux créer des listes à puces (non ordonnées) :

- Liste à puces
- Deuxième élément

Je peux aussi créer des listes numérotées (ordonnées) :

1. Premier élément
2. Deuxième élément
3. Troisième élément

Je peux insérer des liens, comme ce lien vers [GitHub](https://github.com).

Je peux insérer des images :

![Logo GitHub](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)

Je peux insérer des blocs de code :

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Je peux aussi insérer des tableaux (avec alignements) :

| Colonne 1 | Colonne 2 | Colonne 3 |
| :-------- | :-------: | --------: |
| Valeur 1  | Valeur 2  |  Valeur 3 |
| Valeur 4  | Valeur 5  |  Valeur 6 |

> Ceci est un bloc de citation.
>
> Il peut contenir plusieurs lignes.
>
> Il est souvent utilisé pour mettre en avant des citations ou des notes
> importantes.

Il existe aussi des variantes de Markdown, comme
[CommonMark](https://commonmark.org/) ou
[GitHub Flavored Markdown (GFM)](https://github.github.com/gfm/), qui ajoutent
des fonctionnalités supplémentaires ou modifient légèrement la syntaxe.

L'exemple suivant fonctionne sur GitHub :

> [!TIP]
>
> Voici un encadré de type "tip" (conseil) utilisant une extension de Markdown
> spécifique à GitHub.
>
> L'affichage peut varier selon la plateforme utilisée.
>
> Plus de détails ici (en utilisant une autre manière d'insérer un lien avec
> Markdown qui ne demande pas de titre) :
> <https://docs.github.com/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax#alerts>
